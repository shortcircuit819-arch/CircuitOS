# CircuitOS 0.9.1 — The installer actually installs a working app

A critical fix to the 0.9 installer, plus the security hardening from the interim 0.9.0.1 build.

## The installer now ships the whole app

The 0.9.0 `Setup.exe` packaged only the program executable — none of the control panel, starter data, or
overlay files. A fresh install would have launched to an error. **0.9.1 fixes this:** the installer now
bundles the admin UI, the OBS overlay, and a starter catalog, so a clean install runs correctly the first
time.

## Your data now lives in the right place

Installed builds keep your data in `%LocalAppData%\CircuitOS\Data`, **beside** the program rather than
inside it. That folder is untouched by updates — the updater only swaps the program. On first launch the
app seeds it with a starter catalog for the setup wizard. (Settings → About → *Open data folder* opens it.)

## Also in this release

- **Security hardening** (from the interim 0.9.0.1 build): overlay image uploads are validated by their
  real file contents, a Content-Security-Policy protects the control panel, and the dependency set is
  clean.
- **Versioning** moved to standard 3-part `MAJOR.MINOR.PATCH` (e.g. `0.9.1`), which the installer and
  auto-updater require.

## Notes

- Local mode is still the default; upgrading never touches your data.
- Portable ZIP users are unaffected — the ZIP keeps its data folder beside the app as before.
