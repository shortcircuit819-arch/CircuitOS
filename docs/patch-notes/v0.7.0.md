# CircuitOS 0.7.0 — Native Twitch + Cloud Platform

CircuitOS 0.7 is the biggest release since launch. The app now connects directly to
Twitch — channel-point redemptions, chat commands, and pull announcements all flow
through CircuitOS itself with zero Streamer.bot required. Cloud mode (optional) backs
your data to Appwrite so it follows you across machines. Local mode is 100% unchanged
— if you don't touch a Twitch config file or the `--cloud` flag, everything works
exactly as it did in 0.6.

---

## Native Twitch Integration

### Log in once, play forever

A new **Twitch** page in the sidebar lets you sign in with your Twitch account. After
a one-time browser authorization, CircuitOS caches your tokens locally and
auto-refreshes them — no re-login needed mid-stream.

### Channel-point reward management

From the Twitch page you can:
- **Create** a new channel-point reward from your profile's redemption name
- **Attach** an existing reward you already have on your channel
- **Edit** the reward title and point cost through Helix and keep your profile in sync
- **Delete** CircuitOS-managed rewards (attach-only rewards are left untouched)

Reward IDs persist per profile so the listener picks up the right reward on every
restart without re-syncing.

### Redemptions, commands, and chat — built in

Once a profile is live and a reward is attached:
- **Channel-point redemptions** are received over EventSub WebSocket (no public
  endpoint required — the desktop app connects outbound)
- Pulls run through the full shared redemption pipeline: collection selection,
  tier-weighted roll, variant resolution, inventory write, completion detection
- **Pull announcements** are sent to Twitch chat automatically after each fulfillment
- **Chat commands** (`!components`, `!missing`, `!duplicates`, `!balance`, `!leaderboard`,
  salvage) are handled natively — no Streamer.bot paste needed
- The listener starts automatically when CircuitOS launches with a live profile; no
  separate terminal process required

### Multiple live profiles

You can now run more than one game profile simultaneously. Each profile owns its
command words and reward — CircuitOS hard-blocks activation if two live profiles
would share a command or redemption title.

## Admin Panel Improvements

### Overlay editor — per-state color overrides

The Overlay Editor now has a **State Overrides** panel. Switch the preview to Rare,
Complete, or Duplicate and set independent Accent, Label, and Bar Fill colors for each
state. Normal uses the global appearance colors; non-normal states fall back to global
for any field you leave blank.

### Game Profile — one-click save

Saving your Game Profile now also saves any pending catalog changes in the same click.
No more two-step save.

### Eyebrow label removed

The "Eyebrow label" field has been removed from the Game Profile and wizard — it was
wired to a live-preview panel only and never reached the running overlay or Streamer.bot
actions. **Panel nickname** is kept; it drives the sidebar profile widget and view title.

### Commands redesigned

The Commands section in Game Profile is now a clean vertical list — label on the left,
`!` prefix and editable input on the right. The old four-column grid is gone.

### Backups section clarified

Section headers renamed to **"What's covered"** and **"Available backups"** with
simplified guide cards. Empty states now explain what to do rather than showing nothing.

### Overlay slider fill fix

The pull-rate sliders in the Overview and Rate Lab no longer show the colored fill
overshooting past the thumb handle.

### Appearance colors reach navigation

Custom theme colors now drive the sidebar and topbar chrome — the full navigation shell
recolors, not just the content panels.

## Cloud Mode (Optional)

> **Cloud mode is disabled by default.** Pass `--cloud` to opt in.
> Your local setup is completely unchanged without it.

- Data backed to **Appwrite** — catalog, inventory, profile, and boost config all live
  in the cloud; saves write back automatically
- **Rolling recovery backups** — every cloud save snapshots the prior version; restore
  from the Backups view
- **Twitch user ID as tenant** — your cloud data is scoped to your Twitch account

---

## Streamer.bot

Streamer.bot is now **optional**. The native Twitch path handles everything Streamer.bot
used to do. If you prefer to keep using Streamer.bot, it continues to work exactly as
before — the generated actions are still available from the Streamer.bot Setup tab.

---

## ⚠️ Notes for upgraders

- **Twitch login:** you'll need to complete the one-time Twitch authorization from the
  Twitch page before native redemptions go live. Existing Streamer.bot setups keep
  working in the meantime.
- **Scopes:** if you logged in during beta testing and want chat commands, you may need
  to log out and back in once to consent to the newer chat scopes
  (`user:read:chat` / `user:write:chat`).
- **Local data:** no migration needed. Your profiles, inventory, and catalog are
  untouched.
