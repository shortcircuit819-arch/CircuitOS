# CircuitOS v0.5.0 — Profiles and Modules

## What's New

### Multiple game profiles

CircuitOS now supports multiple independent game profiles from a single
installation. Each profile has its own:

- Collection catalog and pull rates
- Viewer inventory
- System profile (branding, terminology, colors, commands, messages)
- Featured boost config
- Overlay config and background image
- Backups

Switch between profiles from the new **Profiles** section in the sidebar. A
confirmation dialog protects against accidental switches during a live stream.

### Automatic data migration

On first launch of 0.5.0, all existing data files are automatically moved into
a `profiles/default/` folder — no manual steps required. The app starts with
the same catalog and inventory you had before.

### Profile management

From the **Profiles** view you can:

- **New Profile** — create an empty profile for a different game
- **Switch** — reload the full admin panel with a different profile's data
- **Rename** — update a profile's display name
- **Delete** — permanently remove a non-active profile

### Module export and import

Export the active profile as a portable `.circuitmodule` file — a JSON bundle
containing the catalog, branding, boost config, role awards, and overlay config.

Import a `.circuitmodule` to create a new profile pre-populated with all of that
data. A useful way to move a game configuration between installations or share a
starter kit.

### IDataStore abstraction layer

All data access now routes through an `IDataStore` interface backed by
`LocalFileDataStore`. This is an internal architecture change with no user-facing
impact, but it prepares the codebase for the 0.7 cloud migration — swapping in
`AppwriteDataStore` will be a single implementation file rather than a rewrite.

## How to Update (Existing Install)

1. Close CircuitOS.
2. Copy from the update package into your install folder:
   - `CircuitOS.exe`
   - `App\` folder
   - `Overlay\` folder
3. Restart CircuitOS.
4. Your existing data will migrate automatically into a `profiles/default/`
   folder on first launch.
5. Go to **Streamer.bot** and regenerate C# actions — the data path has changed
   to include the profile folder. Replace the Execute C# sub-action in
   Streamer.bot with the new generated code.

> **Important:** After updating, the Streamer.bot action path must be updated.
> The old path (e.g. `C:\CircuitOS\Data`) is now
> `C:\CircuitOS\Data\profiles\default`. Regenerate and reinstall the action.
