# CircuitOS 0.6.0.8 — Variant-pull message no longer doubles the label

## Bug Fixed

### "CAFFINATED CAFFINATED Capacitor"

Found during live testing: when the optional **variant-pull** message used both
`{variantLabels}` and `{item}` (e.g. `{viewer} found a {variantLabels} {item}!`),
the variant label appeared twice — *"found a CAFFINATED CAFFINATED Capacitor"*.

**Cause:** in the variant-pull message only, `{item}` was the variant-prefixed
display name ("CAFFINATED Capacitor"), so adding `{variantLabels}` in front
doubled the label.

**Fix:** in the variant-pull message, `{item}` is now the **base** item name
("Capacitor"), so `{variantLabels} {item}` composes cleanly as
"CAFFINATED Capacitor". Every other message is unchanged — their `{item}` still
includes the variant prefix, so `redeemSuccess`, `rarePull`, and `triplePull`
continue to read "CAFFINATED Capacitor" from `{item}` alone.

The Messages editor now documents this on the variant-pull template, and its
live sample preview fills in `{variantLabels}` correctly.

## ⚠️ Action required

This fix is in the Streamer.bot Redemption action. After updating, open
**Streamer.bot Setup**, copy the **Redemption** C# again, paste it over your
installed action, and compile. (If your variant-pull template only uses `{item}`
and not `{variantLabels}`, no message text change is needed — but you still need
the regenerated action for the corrected behavior.)
