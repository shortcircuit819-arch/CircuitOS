# CircuitOS 0.7.0.2 — Cloud from the app + a big clarity pass

A polish-and-usability release on top of 0.7.0.1. The headline: **cloud storage is now
something you turn on from inside CircuitOS** instead of a command-line flag, and the whole app
got a plain-language pass so a brand-new streamer isn't left guessing.

## New: Settings page

CircuitOS now has a real **Settings** page (in the sidebar):

- **Data storage — Local or Cloud.** Local keeps everything on this PC (the default, no account
  needed). Cloud syncs to your **own Appwrite project** so your data follows you across machines.
  Pick a backend, and CircuitOS tells you when a restart is needed to apply it.
- **Appwrite connection** — enter your endpoint, project, API key, and table, then **Test
  connection** before committing. Your API key is write-only in the UI and never shown back.
- **About** — version, current storage mode, and your data folder, with an **Open data folder**
  button.
- **Preferences** — the "Hide System Check card" option now lives here.

> Cloud is an **advanced / self-host** option — most streamers can happily stay on Local, and
> native Twitch already runs fully live in local mode. If cloud ever can't start, CircuitOS
> falls back to local and tells you why, so a bad setting can't lock you out.

## Clearer for new users

A full walkthrough as a first-time, non-technical user drove a wording overhaul:

- **Twitch is now the obvious path.** First-run setup sends you to connect Twitch (not to install
  Streamer.bot), Streamer.bot is marked **optional** in the sidebar, and the save/messages copy no
  longer assumes you're using it.
- **Fixed a misleading claim** that you needed "cloud mode to go live" — native Twitch goes live
  right here in local mode, no extra setup.
- **The setup wizard is friendlier** — every field has an example ("e.g. cards", "e.g. Pull a
  Card"), and it explains what a redemption name is and why singular/plural exist. No more "stable
  IDs are generated automatically."
- **Plain-language help** for Featured Boost, Events (with a nudge that times are UTC), and search.
- **Small polish** — consistent field names, a friendlier reset confirmation, the Twitch
  permissions card reads as onboarding before you log in, clarifying tooltips, and collection names
  no longer get cut off in the health table.

## Under the hood

- Cloud startup is now resilient: it reads your saved backend choice, probes the connection, and
  falls back to local (surfacing the reason) rather than failing to start.

Nothing in your data changes, and local mode is unchanged. Streamer.bot setups keep working.
