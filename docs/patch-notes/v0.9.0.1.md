# CircuitOS 0.9.0.1 — Security hardening

A release-candidate hardening pass on top of 0.9.0. No new features and nothing in your data changes —
this tightens the local control panel against misuse.

## What changed

- **Overlay image uploads are validated by their actual contents.** The server now identifies an uploaded
  background by its real file signature instead of trusting the browser's declared type, so a mislabeled
  or non-image file can't be written as a background.
- **Content-Security-Policy on the control panel.** The admin page now tells the browser to run only its
  own script and block injected inline scripts, external embeds, and page-base hijacking — a backstop
  against cross-site scripting.
- **Dependency audit** — no known-vulnerable or deprecated packages.

This builds on the cross-site request protection added in 0.8.1. The control panel remains bound to your
local machine only.

## Notes

- Local mode is still the default; your data is untouched.
- Verified: the panel, live theming, and all views work unchanged under the new policy.
