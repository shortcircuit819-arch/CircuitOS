# CircuitOS v0.5.0.7 — OBS Overlay Fix

## What's New

### OBS overlay now works in local file mode

The OBS browser source can now be set up using **Local file** mode instead of
an HTTP URL. CircuitOS automatically copies the overlay files into the active
profile's data folder on startup, so the overlay HTML and its state file are
always in the same directory.

On each startup and after every profile switch, CircuitOS writes:

```
Data\profiles\<profile>\overlay\index.html
Data\profiles\<profile>\overlay\overlay.js
Data\profiles\<profile>\overlay\styles.css
```

Streamer.bot already writes `overlay-state.json` to the same folder. With the
files co-located, the local file browser source resolves state updates correctly
without requiring the CircuitOS HTTP server to be reachable from OBS.

### OBS file path shown in Overlay Editor

A new **OBS SETUP** panel at the top of the Overlay Editor displays the exact
local file path for the current profile's overlay. Click **Copy** to put the
path on the clipboard, then paste it into the OBS browser source file picker.

The path updates automatically when you switch profiles.

---

## How to Update (Existing Install)

1. Close CircuitOS.
2. Copy from the update package into your install folder:
   - `CircuitOS.exe`
   - `App\` folder
3. Restart CircuitOS. The overlay files are published to your data folder on
   startup — no manual file moves needed.
4. In OBS, open the browser source properties. Switch from **URL** to
   **Local file** and browse to the path shown in the Overlay Editor.
