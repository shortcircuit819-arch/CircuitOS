# CircuitOS 0.7.0.1 — Native Twitch + Cloud (first 0.7 build)

The biggest release since launch. CircuitOS now talks to Twitch directly — channel-point
redemptions, chat commands, and pull announcements all run through CircuitOS with **no
Streamer.bot required** — and logging in is genuinely one click. Local mode is unchanged; if
you don't connect Twitch or pass `--cloud`, everything works exactly as it did in 0.6.

> Consolidated release note. This supersedes the interim 0.7.0 / 0.7.1 notes (never distributed).

## Native Twitch — zero-config

- **One-click login.** CircuitOS ships with its own Twitch app and logs you in with the Device
  Code Flow: click **Log in with Twitch**, approve in the browser, done. No developer account, no
  client secret, no `twitch.local.json`. The session keeps a refresh token, so it survives long
  streams. Self-hosters can still bring their own app via `twitch.local.json`.
- **Channel-point rewards** managed from the app: create, attach an existing reward, edit
  title/cost, sync, or delete. Reward IDs persist per profile. Per-profile reward cost.
- **Redemptions** over EventSub WebSocket (no hosting needed) run the full pull pipeline and update
  your inventory and the OBS overlay.
- **Chat commands** (`!inventory`, `!missing`, `!duplicates`, `!balance`, `!leaderboard`, salvage)
  handled natively, with per-viewer throttling. **Pull announcements** post to chat automatically.
- The listener **auto-starts** with the app — no separate process.

## Multiple live profiles

Run more than one game profile live at once. CircuitOS blocks activating two live profiles that
share a command word or redemption title so nothing collides.

## Overlay

- **Per-state colors:** the Overlay Editor's State Overrides panel sets Accent/Label/Bar colors for
  Rare, Complete, and Duplicate independently.
- Native redemptions now drive the overlay (previously only Streamer.bot did).

## Admin panel

- New **Command Tester** on the Game Profile page — preview any command's output against a sandbox
  viewer, no Twitch and no live data changed.
- Full UI cleanup: slider rate bars, clickable dashboard cards, unified profile save, clearer
  Backups, redesigned command list, tidier Messages/Collections.

## Cloud mode (optional, `--cloud`)

Back your data to Appwrite — catalog, inventory, profile, boost — with rolling recovery backups,
keyed to your Twitch account. Disabled by default; local mode is untouched without the flag.

## Reliability & security

- Crash-safe atomic inventory writes; native `!salvage` now persists.
- EventSub reconnects on a missed keepalive; duplicate redemptions are de-duped.
- Twitch tokens encrypted at rest (Windows DPAPI). Loopback-only Host-header check on the local API.

## Notable fixes

- Fixed the redeem RNG being seeded so every live pull returned the **same item**.
- Cooldown and duplicate-protection are now read from the profile.
- Appwrite row-addressing desync ("verified 0" / catalog-not-found) resolved.
- Fixed the native listener not starting in the zero-config (no-file) case, which had disabled
  commands, redemptions, and the overlay together.

## Streamer.bot

Now **optional** — native Twitch covers what it did. Existing Streamer.bot setups still work.
