# CircuitOS 0.6.0.6 — Hotfix: Editor crash on tier-enabled collections

## Bug Fixed

### "Cannot access 'hasTiers' before initialization"

0.6.0.5 introduced a crash in the catalog editor. Expanding a collection card,
adding an event, or otherwise rendering a collection body threw a JavaScript
error — **"Cannot access 'hasTiers' before initialization"** — and the affected
panel failed to load.

**Cause:** the new bulk-assign toolbar (0.6.0.5) referenced the `hasTiers`
variable a few lines *above* where that variable was declared. JavaScript's
temporal dead zone makes reading a `const` before its declaration a hard error,
so every collection-card render threw before it could draw the items list.

**Fix:** the `hasTiers` declaration now sits above its first use, and the
duplicate declaration further down was removed. Bulk assign, the per-tier
"← Unassigned" button, and the CSV tier importer all work as intended.

If you installed 0.6.0.5, update to 0.6.0.6 — no data changes, EXE + admin
panel only. No Streamer.bot action regeneration required for this fix.
