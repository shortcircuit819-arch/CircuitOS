# CircuitOS 0.7.2 — Streamer.bot retired; native Twitch is the one path

CircuitOS started as a Streamer.bot add-on because Twitch integration had to live somewhere. Since
0.7.0 that reason is gone: native Twitch does redemptions, chat commands, reward management, and the
overlay with a one-click login and no code to paste. 0.7.2 finishes the transition by removing the
Streamer.bot integration entirely, so there is one clear way to go live.

## What changed

- **Streamer.bot integration removed.** The Streamer.bot Setup page, the generated C# actions, and the
  bundled action templates are gone. Native Twitch is the single supported path.
- **Nothing about your game changes.** Catalog, profiles, inventory, overlay, messages, commands,
  salvage, events, and boosts are untouched. Local mode is still the default.
- **Cleaner app and docs.** The setup screen and its scattered "regenerate your actions" reminders are
  gone; the setup guides now describe connecting Twitch.

## If you were using Streamer.bot

Connect Twitch from the **Twitch** section (one-time login, no developer account or config files).
CircuitOS creates and manages the channel-point reward, then handles redemptions, chat commands, and
pull announcements directly through EventSub. You can delete your old CircuitOS Streamer.bot actions.

## Notes

- No data migration. Your Data folder is unchanged.
- The legacy PowerShell admin (`CircuitAdmin.ps1`) remains only as an emergency fallback and is not
  part of the shipped app.
