# CircuitOS 0.6.0.3 — Rolling Logic, Variants & Tiers Live

## What's New

### Tier-weighted item pulls

When a collection has Rarity Tiers configured, redeems now pull items using weighted
tier odds instead of flat equal odds. The pull works in two steps:
1. Roll the tier (e.g., Common 70 / Rare 25 / Ultra Rare 5).
2. Pick a random item from within that tier.

Collections without tiers continue to pull all items at equal odds — no change.

Duplicate protection interacts correctly: if all items in a given tier are already
owned, that tier is excluded from the roll for that viewer.

### Variant tags on pulls

When a collection has Variants configured, each pull now independently rolls each
variant and applies up to two tags to the result:
- `{item}` in all chat messages automatically includes the variant prefix
  (e.g., "SHINY Bulbasaur", "LARGE SHINY Bulbasaur").
- The overlay displays variant tags in the tags row.
- A tier label also appears in the overlay tags row when tiers are active.

### New optional message: Variant Pull

A new **Variant Pull** message template is available in Message Templates. It fires
as an extra chat line whenever one or more variants land on a pull. Leave it blank
(the default) to disable it entirely.

Available placeholders: `{variantLabels}`, `{viewer}`, `{item}`, `{collection}`.

Example: `✨ {variantLabels} PULL: @{viewer} just pulled {item}!`

## Important: Regenerate Streamer.bot Action

The redemption action has been updated with tier-weighted selection and variant
rolling logic. Paste the freshly generated **Redemption** action code from the
Streamer.bot Setup tab and recompile.

## Overlay State Changes

`overlay-state.json` now includes two new fields written by the redemption action:
- `variantLabels` — array of variant tag strings that fired (may be empty).
- `tierLabel` — the label of the tier that was rolled (empty if no tiers).

Older overlay state files without these fields are handled gracefully.
