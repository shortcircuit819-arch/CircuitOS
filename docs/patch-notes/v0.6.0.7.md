# CircuitOS 0.6.0.7 — Tier editing polish

## Bug Fixed

### Renaming a tier ID no longer orphans its items

Previously, editing a rarity tier's **ID** in the collection editor left every
item still pointing at the *old* tier ID. Because tiered collections require
every item to belong to a valid tier, the next **Save Catalog** failed
validation and you had to reassign the affected items by hand — painful on a
large catalog.

Renaming a tier ID now migrates every item assigned to that tier automatically,
the same way renaming a collection key already migrates its boost reference. The
item-to-tier assignments stay intact.

## Polish

- The Rate Lab tier breakdown bar no longer computes an invalid width for a
  tiered collection whose effective pull rate is zero (e.g. weight 0 or an
  inactive event). It now renders an empty bar instead.

This release is part of the 0.6.x consolidation pass — a code audit of the
0.6 variant/tier paths before live integration testing. The Streamer.bot
rolling logic and OBS overlay rendering were reviewed and need no changes.
No data changes; EXE + admin panel only.
