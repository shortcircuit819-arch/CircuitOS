# CircuitOS v0.4.2 — Overlay Editor Fixes

## What's New

### Overlay files now update with every release

The overlay assets (`overlay.js`, `styles.css`, `index.html`) are now shipped
in an `Overlay\` folder at the install root instead of inside `Data\overlay\`.
The runtime prefers `Overlay\` first, so copying the update package always
refreshes the overlay. Previous updates silently left the old overlay.js in
place because the update package never touched `Data\`.

If you are updating from 0.4.0 or 0.4.1, copy the `Overlay\` folder from the
update package into your install folder alongside `CircuitOS.exe`.

### Overlay editor changes are now captured reliably

Color pickers previously required you to close the picker AND click away before
the value was registered. The editor now listens to `input` events, so any
change you make in the color picker is captured immediately when the Save button
is clicked.

### Background image restored

The in-app default overlay config was missing `backgroundImage`, so the field
appeared empty after a fresh install and the value was dropped on the first
save. Fixed — the field now initialises correctly and saves as expected.

### Overlay preview no longer constantly refreshes

This was caused by the old `overlay.js` (from a pre-0.4.1 install) running its
500 ms state-polling loop even in preview mode. The new `overlay.js` shipped in
`Overlay\` disables polling when loaded with `?preview=1`.

### Sidebar — Settings moved to bottom, Overlay Editor inside Settings

The Settings group (Branding, Overlay Editor, Messages) is now at the bottom
of the nav, below Tools. Overlay Editor sits inside Settings alongside Branding
rather than as a standalone top-level item.

---

## How to Update (Existing Install)

1. Close CircuitOS if it is running.
2. Copy these files from the update package into your install folder:
   - `CircuitOS.exe`
   - `App\` folder (all files)
   - `Overlay\` folder — **new, required to fix the overlay editor**
3. Restart CircuitOS.
