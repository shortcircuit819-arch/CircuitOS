# CircuitOS v0.5.0.8 — Background Image Fix

## What's New

### Custom background image restored in local file mode

The overlay background image now loads correctly when OBS uses Local file mode.

**What was broken:** Older versions stored the background image reference as
`/overlay-bg` — an HTTP endpoint path. In Local file mode (`file://`), that
path resolves to nothing, so the background disappeared after the 0.5.0.7
update.

**What changed:**
- New uploads now store the image as a relative filename (`bg.png`, `bg.jpg`,
  etc.) instead of the HTTP endpoint path. This resolves correctly in both
  Local file mode and HTTP mode.
- The HTTP server now also serves `bg.*` directly under `/overlay/bg.*`, so
  the admin panel preview and HTTP-mode OBS sources continue to work.
- Existing configs that still have the old `/overlay-bg` path are automatically
  remapped to `bg.png` when the overlay runs in Local file mode, so the
  background reappears without any action from you.

---

## How to Update (Existing Install)

1. Close CircuitOS.
2. Copy from the update package into your install folder:
   - `CircuitOS.exe`
   - `App\` folder
   - `Overlay\` folder
3. Restart CircuitOS.

No re-upload of the background image is needed — the existing file is already
on disk at the correct location.
