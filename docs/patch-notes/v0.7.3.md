# CircuitOS 0.7.3 — Collection packs

Share one collection, not your whole game. 0.7.3 adds **collection packs** — a way to hand someone a
single collection that drops into their setup wearing *their* theme — plus a fix so imports never
create confusing duplicate profiles.

## Collection packs (`.circuitcollection`)

Every collection now has a **Share** button (in the Collections editor). It exports a
`.circuitcollection` file carrying that one collection plus its gameplay flavor — commands, chat
messages, rare labels, terminology, and tuning (cooldown, dup-protection, cost) — but **not** your
colors.

Import one from **Profiles → Import Module / Pack**. It becomes a **new profile that adopts your
current theme** (colors, brand, overlay), so a shared collection looks like *your* game, not the
sharer's. You name it on import (pre-filled with the sharer's game name), and if any of its commands
clash with a profile you already have live, you just rename them before going live.

Think of a pack as a **themeable mini-game**: it brings the content and flavor and wears your skin.
(A `.circuitmodule` is still the whole-profile export — every collection plus the sharer's branding.)

Collection packs are plain files you share directly — no cloud or account needed.

## Import name de-duplication

Importing a profile or pack whose name already exists no longer creates a second, indistinguishable
copy — the new one is auto-suffixed (e.g. `Circuit Components (2)`), so your profile list stays clear.
