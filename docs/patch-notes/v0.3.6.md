# CircuitOS v0.3.6 Patch Notes

## Bug Fixes

**Featured boost no longer shows on unrelated pulls**
When a featured stream boost was active, the boost name was appearing on every
pull message — even pulls from collections that weren't part of the boost. It
now only appears when the rolled collection actually has a boost multiplier.

**Streamer.bot tab version now matches the app version**
The integration version displayed on the Streamer.bot setup tab was showing an
old internal build number instead of the current CircuitOS version. It now
reads the correct version number.

---

## How to Update

**Existing install (update package):**
1. Close CircuitOS if it is running.
2. Replace `CircuitOS.exe` in your install folder with the one from the update package.
3. Restart CircuitOS.
4. On the Streamer.bot tab, click **Generate** to get a fresh copy of the
   Redemption action with the boost fix applied.
5. Paste the updated Redemption action into Streamer.bot and recompile.

No data files are changed by this update. Your inventory, catalog, and profile
are untouched.

**Event collection CSV import**
Event collections now have an **Import Items** button on the Events tab, matching
the same workflow as the regular collection importer. Paste a names-only list or
load an `id,name` CSV, set the pull weight, salvage value, and event window, then
apply to the editor. The new event starts disabled — enable it when you're ready
to go live.
