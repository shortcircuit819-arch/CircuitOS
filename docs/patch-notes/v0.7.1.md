# CircuitOS 0.7.1 — Overlay images, cloud settings, and a big polish pass

0.7.1 rolls up the 0.7 milestone into a genuinely polished build: native Twitch that just works,
an optional cloud backend you can turn on from the app, per-state overlay artwork, and a top-to-bottom
usability pass so a brand-new streamer is never left guessing. Local mode is still the default and
unchanged.

## New in 0.7.1

### Per-state overlay images
Upload a different background image or GIF for each pull state — a celebratory GIF when a collection
**completes**, a special look for a **rare** pull, something distinct for a **duplicate**. Set them in
the Overlay Editor's State Overrides panel (switch to a state, then upload its background). Any state
you don't customize simply uses your global background, so you only add art where you want it.

### Automatic backup cleanup
Config backups used to pile up forever. CircuitOS now keeps the most recent set per file type
(30 by default) and trims the rest automatically — your recovery points stay useful without the folder
growing without bound. Viewer inventory backups are never touched.

## Also in the 0.7 line (recap)

### Native Twitch — zero-config
One-click Twitch login (no developer account, no config files), CircuitOS creates and manages the
channel-point reward, and redemptions, chat commands, and pull announcements all run directly through
EventSub. No code to paste. Streamer.bot remains an optional alternative.

### Cloud, from the app
A Settings page lets you turn on an optional cloud data backend (your own Appwrite project) with a
Test-connection step and a safe fallback to local if it can't start. Most streamers can stay on Local.

### Clearer for new users
First-run now points you at connecting Twitch (not installing Streamer.bot), the wizard has real
examples on every field, and jargon around boosts, events, and cloud was rewritten in plain language.

### Overlay & reliability
Per-pull-state colors (rare/complete/duplicate), native redemptions driving the overlay, crash-safe
atomic inventory writes, EventSub keepalive + reconnect, per-viewer chat throttle, and DPAPI-encrypted
Twitch tokens.

## Notes
- Nothing in your data changes; local mode is unchanged.
- Fixed a subtle overlay bug where per-state colors could linger from a previous pull — state styling
  now always resets correctly.
