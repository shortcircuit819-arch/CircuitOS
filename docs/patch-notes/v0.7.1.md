# CircuitOS 0.7.1 — One-click Twitch login (zero-config)

The headline of 0.7 was native Twitch. 0.7.1 makes **logging in** match that promise:
no Twitch developer account, no app registration, no config files.

## Just click "Log in with Twitch"

Previously, connecting Twitch meant every streamer had to register their own Twitch
developer application and drop a `twitch.local.json` with a client ID and secret into the
data folder. That's gone.

Now CircuitOS ships with its own Twitch application built in, and login uses Twitch's
**Device Code Flow**:

1. Click **Log in with Twitch**.
2. Your browser opens to Twitch's authorize page (the code is already filled in).
3. Click **Authorize** — done.

No file to create, no secret to paste, no developer account. The session also keeps a
**refresh token**, so it renews itself and won't drop in the middle of a long stream.

### Why this matters

- **Distributable.** A client secret can't be safely shipped inside a desktop app, which is
  why the old model required each streamer to bring their own. The device flow needs only the
  public client ID, so the same build works for everyone with zero setup.
- **Self-hosters still supported.** If you'd rather use your own Twitch app, drop a
  `twitch.local.json` with your `clientId` (and optionally `clientSecret`) and CircuitOS uses
  it instead — the bundled client ID is just the default.

## Notes

- Existing logins keep working; your cached token is unaffected.
- If you previously created a `twitch.local.json` with a client secret, CircuitOS will keep
  using that (legacy) flow. Remove the file to switch to the zero-config built-in login.
