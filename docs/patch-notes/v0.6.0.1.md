# CircuitOS 0.6.0.1 — Item Variants (Catalog Layer)

## What's New

### Item Variants — catalog editor support

Collections now support an optional **Variants** section in the catalog editor. Variants let
you define cosmetic tags that can randomly fire on a pull — SHINY, FOIL, SLEEPY, LARGE, or
anything else you name.

**How to configure variants:**
1. Open any collection in the Permanent or Event Collections editor.
2. Scroll past the items list to the new **Variants (optional)** section.
3. Click **Add Variant** and fill in three fields:
   - **ID** — internal slug (e.g., `shiny`)
   - **Label** — what appears in chat and the overlay (e.g., `SHINY`)
   - **Chance (0–1)** — probability per pull (e.g., `0.05` = 5%)
4. Save the catalog.

**Design rules:**
- Variants are defined per collection and apply to every item in that collection.
- Each pull rolls variants independently after the item is selected.
- Up to **two** variant tags can fire on a single pull (e.g., LARGE SHINY Bulbasaur).
- Owning any variant of an item counts as owning the base item for duplicate and
  collection-completion purposes.
- Catalogs without variants are fully unaffected — this field is optional.

The rolling logic and overlay display for variants will land in **0.6.0.2**.

## Technical

- `ValidateConfiguration` now validates the `variants` array on each collection: IDs must
  follow the slug format, labels are required, and chance must be strictly between 0 and 1.
- `serializeModel` omits the `variants` key if the array is empty (clean output).
- Patch-note generator tracks variant add / remove / change as diff entries.
