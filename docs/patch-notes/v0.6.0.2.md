# CircuitOS 0.6.0.2 — Rarity Tiers (Catalog Layer)

## What's New

### Rarity Tiers — catalog editor support

Collections now support an optional **Rarity Tiers** section in the catalog editor. Tiers let
you give items within a collection different pull weights — Common, Rare, Ultra Rare, or any
names you choose.

**How to configure tiers:**
1. Open any collection in the Permanent or Event Collections editor.
2. Scroll to the new **Rarity Tiers (optional)** section.
3. Click **Add Tier** and fill in three fields:
   - **ID** — internal slug (e.g., `rare`)
   - **Label** — display name (e.g., `RARE`)
   - **Weight** — relative pull weight (e.g., `25`)
4. Once tiers are defined, each item row gains a **Tier** dropdown — assign every item.
5. Save the catalog.

**Design rules:**
- Tiers are optional. Collections without tiers pull all items at equal odds (existing behavior).
- When tiers exist, **every item must be assigned** to a tier — saving with unassigned items
  is blocked by validation.
- Removing all tiers from a collection clears tier assignments from every item automatically.
- Tier pull logic (weighted roll → random item within that tier) ships in **0.6.0.3**
  alongside variant rolling.

## Technical

- `ValidateConfiguration` validates the `tiers` array (id slug format, label required,
  weight > 0, unique ids) and enforces that every item has a valid tier assignment when
  tiers are defined.
- `serializeModel` omits the `tiers` key if the array is empty.
- Item rows grow a Tier column in the UI when tiers exist; removing all tiers strips
  `tier` from all part objects.
- Patch-note generator tracks tier add / remove / weight change as diff entries.
