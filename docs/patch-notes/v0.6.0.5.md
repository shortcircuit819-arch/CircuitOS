# CircuitOS 0.6.0.5 — Bulk Tier Assignment & CSV Tier Import

## What's New

### Bulk tier assignment toolbar

Assigning tiers to large catalogs (like a 1000-item Pokémon set) no longer
requires editing each item individually. When a collection has tiers defined,
a **Bulk assign** toolbar appears above the items list:

- **Tier dropdown** — choose the tier to assign
- **Assign all** — sets every item in the collection to the chosen tier in one
  click; use this to get everything into Common first, then flip exceptions
- **Assign unassigned** — assigns only items that have no tier yet (or an
  invalid tier ID), leaving already-assigned items untouched

Each tier row in the Rarity Tiers section also now has an **← Unassigned**
button that assigns all tier-less items directly to that tier, letting you
build up tier groups from the tier list rather than the toolbar.

### CSV importer — tier column support

The bulk names/CSV importer now recognizes a `tier` column in header rows:

```
id,name,tier
bulbasaur,Bulbasaur,common
charizard,Charizard,rare
mewtwo,Mewtwo,ultra_rare
```

- The tier value is passed through to each imported item as its `tier` field
- When importing into an **existing** collection with tiers, unrecognized tier
  IDs are flagged with a CHECK warning in the preview — you can still apply
  and fix them in the editor
- When importing into a **new** collection, tier values are carried through
  and will take effect once you define matching tiers on the collection
- The import preview table gains a **Tier** column when tier data is detected

Combined with "Assign all → Common" for bulk baseline, then targeted CSV
imports for rarer items, this covers the full range of catalog setup workflows.
