# CircuitOS 0.8.1 — Bot account, per-profile overlays, security fix

Two requested features and one under-the-hood security fix, on top of the 0.8 design release.

## Bot chat account (optional)

You can now connect a **dedicated bot account** so CircuitOS's chat replies and pull announcements post
**as the bot** (e.g. `@yourbot`) instead of from your own channel account. Find it on the **Twitch**
page — a "Bot chat account" card appears once your main account is connected.

- Click **Connect Bot Account**, then in a **private/incognito browser window** sign in to Twitch as the
  bot account and enter the code shown.
- Once connected, every `!command` reply and pull announcement comes from the bot. Everything else —
  channel-point rewards, redemptions, reading chat — keeps running on your main account.
- Not connected? Nothing changes: replies post from your channel exactly as before.
- If bot messages don't show up, refresh your **main** login once (it grants the bot permission to post),
  or make the bot a moderator.

## Per-profile OBS overlays

Running more than one profile live at once? Each **live profile now has its own OBS overlay**. On the
**Manage Profiles** page, every live profile's card shows its own **OBS browser-source path** with a
**Copy** button — so you can point a separate OBS source at each game. Previously only the profile you
were editing had a working overlay file.

## Security

Hardened the local control-panel server against cross-site request forgery: a malicious webpage can no
longer send "drive-by" commands to CircuitOS running on your PC. Requests from any non-local web origin
are now rejected. This is invisible in normal use — the admin panel and OBS are unaffected.

## Notes

- Nothing in your data changes. Local mode is still the default.
- The bot account's tokens are encrypted at rest on your PC, same as your main login.
