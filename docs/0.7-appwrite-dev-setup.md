# 0.7 — Appwrite Dev Setup (quick)

Stand up Appwrite and create the one collection CircuitOS needs for **model A**
(document-per-key, see `0.7-cloud-foundation.md`).

**Use Appwrite Cloud** (recommended — no Docker, nothing to install locally). The
self-hosted Docker path is kept at the bottom as an alternative, but it is **not
required** and won't work on machines that can't run Docker/virtualization.

The CircuitOS code is the same either way — only the **endpoint** differs. Cloud
also matches the 0.7 end state (a hosted platform), so it's the more realistic dev
target.

## 1. Create an Appwrite Cloud account + project

1. Go to **https://cloud.appwrite.io** and sign up (free tier is plenty for dev).
2. **Create project** → name it `circuitos-dev`. Note its **Project ID**.
3. The **API Endpoint** is `https://cloud.appwrite.io/v1` (Settings confirms it).

That's the entire "install" step — no Docker, no local server.

## 2. Create the database + collection (model A)

1. **Databases** → Create database → id `circuitos`.
2. Inside it → Create collection → id **`profile_data`**.
3. Add **attributes** (Collection → Attributes):

   | Key | Type | Size / notes | Required |
   |-----|------|--------------|----------|
   | `userId` | String | 64 — tenant (Twitch user id) | yes |
   | `profileId` | String | 64 | yes |
   | `dataKey` | String | 32 — one of: catalog, boost, inventory, roles, profile, overlay-config | yes |
   | `json` | String | **large** (set max size high, e.g. 1,000,000) — the document blob | yes |

   *(Appwrite gives every document `$id`, `$createdAt`, `$updatedAt` for free.)*

4. Add an **index** (Collection → Indexes): a **unique** key index on
   `userId`, `profileId`, `dataKey` (ascending). One row per (tenant, profile,
   data file) — the lookup/upsert key.

5. **Permissions** (Collection → Settings): for dev, enable document security and
   allow the API key to read/write. Real per-user permissions arrive with auth in a
   later phase.

> **Size note:** a single Appwrite document/attribute has limits. `inventory` is the
> file most likely to grow large; if it bumps the string-size ceiling, that's the
> signal to move `inventory` to model B (per-viewer documents), as the foundation
> doc anticipates. Fine as a large string for dev.

## 3. Create a dev API key

1. Project → **Settings → API keys** (or Integrations) → Create API key
   (`circuitos-dotnet-dev`).
2. Scopes: at least `databases.read`, `databases.write`, `documents.read`,
   `documents.write`. (Add `files.*` later for the overlay-background bucket.)
3. Keep the secret local — **do not commit it.** `AppwriteDataStore` will read
   endpoint + project id + key from local config / environment, never from source.

## 4. Connection config (where the key goes)

`AppwriteDataStore` reads the connection from a **local JSON file** (recommended on
Windows — env vars need the right scope and an app restart). Copy
`appwrite.local.example.json` (repo root) to **`appwrite.local.json`** and place it
in your **CircuitOS Data folder root**
(`C:\Users\nicho\Documents\CircuitOS\Data\appwrite.local.json`). Format:

```json
{
  "endpoint": "https://cloud.appwrite.io/v1",
  "projectId": "your-project-id",
  "apiKey": "your-secret-api-key",
  "databaseId": "circuitos",
  "collectionId": "profile_data"
}
```

`appwrite.local.json` and `.env` are gitignored — the real key never enters source
control. **Don't paste the `apiKey` into chat** either; just tell me the endpoint and
project id.

**Env-var alternative** (if you prefer): the runtime will also read these process
environment variables, which override the file when set —

```
CIRCUITOS_APPWRITE_ENDPOINT       = https://cloud.appwrite.io/v1
CIRCUITOS_APPWRITE_PROJECT_ID     = your-project-id
CIRCUITOS_APPWRITE_API_KEY        = your-secret-api-key
CIRCUITOS_APPWRITE_DATABASE_ID    = circuitos
CIRCUITOS_APPWRITE_COLLECTION_ID  = profile_data
```

(A `.env` file is just a handy place to keep those for reference — the app reads real
environment variables, not the file, unless you export them.)

## 5. (Later, not now) Storage + hosted auth

## Alternative: self-hosted via Docker (optional, NOT required)

Only if you have a machine that can run Docker (e.g. a Linux box or a cheap VPS —
**not** needed for dev, and skip this entirely if your PC can't run Docker). Steps
2–4 above are identical; only the endpoint becomes your server's address.

In a folder like `C:\Appwrite`, PowerShell:

```powershell
docker run -it --rm `
    --volume /var/run/docker.sock:/var/run/docker.sock `
    --volume ${pwd}/appwrite:/usr/src/code/appwrite:rw `
    --entrypoint="install" `
    appwrite/appwrite:1.8.1
```

Needs ~2 CPU / 4 GB RAM / 2 GB swap and Docker Compose v2. A cheap cloud VM
(DigitalOcean/Linode marketplace has one-click Appwrite images) is the usual route
if you want self-hosted data control later — but for building 0.7, **Appwrite Cloud
above is simpler and sufficient.**
