# 0.7 — Twitch Login

As of **0.7.1**, connecting Twitch is **zero-config**: CircuitOS ships with its own Twitch
application and logs you in with the **Device Code Flow**. There's nothing to register and no
file to create.

## The default: just log in

1. Click **Log in with Twitch** (or run `--twitch-login`).
2. Your browser opens to Twitch's authorize page with the code already filled in.
3. Click **Authorize** — done. Your identity + tokens are cached (encrypted) in
   `twitch-tokens.local.json`.

No Twitch developer account, no client secret, no `twitch.local.json`. The session keeps a
**refresh token**, so it renews itself and survives long streams.

### Why a bundled client ID is safe

An OAuth **client ID is public** — it appears in every authorization request. A **client
secret** is not, and can't be safely shipped inside a desktop app (anyone could extract it),
which is why the old model made each streamer bring their own app. The Device Code Flow needs
only the public client ID, so one build works for everyone. The bundled CircuitOS Twitch app is
registered with **Client Type = Public** and the device-code grant enabled.

## Scopes requested at login

- `channel:read:redemptions` — read channel-point redemptions
- `channel:manage:redemptions` — create/manage the channel-point reward
- `user:read:chat` — listen for chat commands
- `user:write:chat` — send pull announcements and command replies

You'll see these on Twitch's consent screen. If you logged in before a scope was added, log in
again so Twitch can ask for the new permission.

## Advanced: bring your own Twitch app (optional)

Self-hosters can override the bundled client ID. Create **`twitch.local.json`** in your
CircuitOS **Data folder**:

```json
{
  "clientId": "your-twitch-client-id"
}
```

- **Client ID only** (no secret): CircuitOS uses the Device Code Flow against *your* app.
  Register it at https://dev.twitch.tv/console/apps with **Client Type = Public**.
- **Client ID + `clientSecret`**: CircuitOS uses the legacy loopback authorization-code flow
  (register with **Client Type = Confidential** and add OAuth Redirect URL
  `http://localhost:8765`). Only needed if you specifically want the secret-based flow.

`twitch.local.json` and the cached `twitch-tokens.local.json` are gitignored. Tokens are
encrypted at rest with Windows DPAPI (CurrentUser scope). Don't paste a client secret into chat.

## Cloud tenant note

When `--cloud` is used, your authenticated **Twitch user id** becomes the Appwrite tenant
(replacing the `local-dev` placeholder), so cloud data is keyed to the real streamer. A one-time
`local-dev → Twitch-id` migration runs on first cloud start after login.
